# 数据蒸馏技术

- 基于模型：生成对抗网络GAN或变分自编码器VAE
- 基于采样：核心集选择（Coreset Selection）
- 基于优化：优化合成数据特征



### 核心集选择（或者实例选择）

核心集选择：直接从原始的数据集中选择最具代表性或者最有价值的样本，保留原始数据集中的代表性的数据，其他的数据删除。

代码库：DeepScore

核心集选择方法：

#### 一、基于几何的方法

根据特征空间的距离来判断数据的相似性，去除冗余信息

1、**Herding算法**：根据核心集中心和原始数据集中心之间的距离选择数据点，每次一贪婪的方式讲一个样本逐步加入到核心集中。

步骤：

- 初始化一个空的核心集
- 每次在未选择的样本中选择一个最优的样本加入到核心集中，使得核心集的中心点与原始数据集中心点之间的距离最小化，然后更新核心集
- 一直迭代到满足终止条件，即终止算法

缺点：容易造成核心集的样本全部都分散在原数据集中心点附近。

2、**k-Center Greedy算法**：试图解决最小最大设施选址问题。

步骤：

- 随即或者选择一个离原始数据集中心最远的样本加入到核心集
- 计算未选择样本与已选择的所有样本的最近距离，再在未选择样本中的最近距离里面选择距离最远的样本。比如说，现在已选择的样本有三个a,b,c，未选择的样本有两个d，e，先计算d距离a，b，c的最近距离作为d距离已选样本的距离，再计算e距离a，b，c的最近距离作为e距离已选样本的距离，再比较d，e和已选样本的距离，选择最大的那个放入到已选样本中。
- 一直迭代到满足终止条件。

**k-Center Greedy算法和Herding算法的区别**：

k-Center Greedy算法希望选出的样本能覆盖所有的样本，目的是不会遗漏数据

Herding算法希望选出的样本与整体样本分布相似，目的是更像整体

#### 二、基于不确定性的方法

置信度低的样本对模型优化的影响可能更为显著。优先挑选置信度低的样本加入到核心集中

#### 三、基于误差/损失的方法

- 根据样本对误差和损失的贡献程度来判断样本的重要性。
- 根据样本在训练过程中遗忘（即上一轮训练中已被正确分类，在本轮分类错误）的次数来判断样本的重要性。

#### 四、基于决策边界的办法

分布在决策边界附近的数据难以区分，因此接近决策边界的数据也可以用作核心集

- 对抗性DeepFool：如果无法获得准确的决策边界距离，通过对样本施加扰动，知道样本预测标签发生改变，哪些需要最小对行性扰动的数据点最接近决策边界
- 对比主动学习：为了找到决策边界附近的数据，通过计算样本点的预测似然与邻居的差异，选择最大的样本构建核心集

#### 五、基于梯度匹配的方法

完整的训练数据集所产生的梯度，可以被子集产生的加权梯度所替代，通过最小化子集与全集在模型参数空间上的梯度差异，选取最具代表性的数据点

#### 六、基于双层优化的方法

核心集选择可以看成是一个双层优化的问题，将子集选择作为外层目标，将模型在这个子集上的优化作为内层目标。



# 图像去重

python去重库：imagededup

**imagededup库图像编码方式：**

1、哈希编码：64位哈希值，表示为16字符的十六进制字符串

2、CNN编码：使用CNN来提取图像特征（内置的有MobilenetV3、ViT、EfficientNet）



**哈希算法：**

1、PHash：感知哈希算法

步骤：

- 将图片resize为32X32
- 将rgb图像转成为灰度图像
- 使用DCT（离散余弦变换）将图像从空间域转换到频率域
- 提取DCT左上角的8X8的低频区域（图像的信息集中在低频区域）
- 计算8X8的低频区域的平均值
- 将每个系数与平均值比较，大于平均值就设置为1，否则为0

2、AHash：平均哈希算法（相似的图像会具有相似的哈希值）

步骤：

- 将图片resize为8X8像素
- 将rgb图像转成为灰度图像
- 计算灰度图像的平均值
- 遍历每个像素点，大于平均值设为1，否则设置为0
- 由此生成一个64为的二进制数（图像的平均哈希值）

3、DHash：差异哈希算法

步骤：

- 将图片resize为8X8像素
- 将rgb图像转成为灰度图像
- 从第一行的第一个像素开始，比较每个像素与其右侧像素的亮度差异，大有右侧像素设为1，否则为0。
- 生成64为的二进制数

4、WHash：小波哈希算法

步骤和感知哈希相似，只是将感知哈希算法里面的DCT换成DWT（离散小波变换）。
